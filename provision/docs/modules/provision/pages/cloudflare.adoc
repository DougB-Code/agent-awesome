= Provisioning Cloudflare Resources
:description: Cloudflare resources managed by the provisioning CLI.

The provisioner reconciles one isolated cloud agent per `agent-id`.
Resource names are derived from the stable agent id and must be Cloudflare-compatible.

== Managed Resources

[cols="1,2",options="header"]
|===
|Resource |Managed behavior

|R2 bucket
|Create, detect existing buckets, and delete during confirmed cleanup.

|Worker secrets
|Upload gateway, persistence, model, and optional Slack secrets.

|Worker route
|Validate, create, repair, and remove route bindings.

|DNS readiness
|Check that expected hostnames can route to the Worker.

|Container deployment
|Use Wrangler for build and deployment while direct API support remains impractical.
|===

Generated artifacts belong under `build`.
They must not contain secret values.
