= Cloudflare Deployment
:description: Cloudflare Worker and Container deployment documentation.

The Cloudflare deployment scaffold lives in `deploy/cloudflare`.
It hosts the personal Slack pilot using one Worker in front of one Container.

== Responsibilities

* Route Slack-signed event requests to the gateway.
* Route public liveness checks to `/healthz`.
* Reject unrelated public requests at the Worker boundary.
* Require bearer auth for `/api/*` and `/mcp` before traffic reaches the container.
* Run the gateway, harness, and memory binaries inside the container.
* Restore and save memory snapshots through a private Worker endpoint.
* Keep deployment configuration separate from local harness and UI behavior.

== Files

[cols="1,2",options="header"]
|===
|Path |Use

|`../../Dockerfile.cloudflare`
|Builds gateway, harness, and memory binaries into one Linux container.

|`scripts/entrypoint.sh`
|Starts the gateway with harness and memory auto-start flags.

|`config/*.yaml`
|Pilot model, agent, and tool configuration used in the container.

|`worker/src/index.ts`
|Worker, Container class, request routing, and persistence endpoints.

|`worker/wrangler.jsonc`
|Container, Durable Object, route, R2 binding, vars, and secrets declaration.
|===
