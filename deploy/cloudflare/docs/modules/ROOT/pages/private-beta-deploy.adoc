= Private Beta Cloudflare Deploy
:description: Cloudflare-specific deployment checklist for the private beta pilot.

This page covers the single Cloudflare Worker and Container pilot.
For per-agent provisioning, use xref:operations:private-beta-runbook.adoc[].

== Required Files

[source,sh]
----
test -f Dockerfile.cloudflare
test -f deploy/cloudflare/worker/src/index.ts
test -f deploy/cloudflare/worker/scripts/smoke-test.mjs
----

== Local Verification

[source,sh]
----
npm --prefix deploy/cloudflare/worker test
npm --prefix deploy/cloudflare/worker run check
----

The smoke test verifies Worker routing, required secrets, Container binding, R2 snapshot binding, and Slack signature forwarding without deploying.

== Secrets

[source,sh]
----
cd deploy/cloudflare/worker
GATEWAY_TOKEN="$(openssl rand -base64 32 | tr '+/' '-_' | tr -d '=')"
PERSISTENCE_TOKEN="$(openssl rand -base64 32 | tr '+/' '-_' | tr -d '=')"
printf '%s\n' "$GATEWAY_TOKEN" | npx wrangler secret put AGENTAWESOME_GATEWAY_TOKEN
printf '%s\n' "$PERSISTENCE_TOKEN" | npx wrangler secret put AGENTAWESOME_PERSISTENCE_TOKEN
npx wrangler secret put SLACK_SIGNING_SECRET
npx wrangler secret put SLACK_BOT_TOKEN
npx wrangler secret put OPENAI_API_KEY
----

Set all three Slack allow-list vars in `wrangler.jsonc` before deploying.

== Deploy

[source,sh]
----
cd deploy/cloudflare/worker
npx wrangler r2 bucket create agent-awesome-beta-context
npx wrangler deploy
----

Set the Slack Events API request URL to:

[source,text]
----
https://agent-awesome.com/slack/events
----

Check the authenticated beta status surface after deploy:

[source,sh]
----
curl -fsS \
  -H "Authorization: Bearer ${GATEWAY_TOKEN}" \
  https://agent-awesome.com/api/gateway/beta-status
----

== Snapshot Check

[source,sh]
----
mkdir -p ../../../build/ai
curl -fsS \
  -H "Authorization: Bearer ${PERSISTENCE_TOKEN}" \
  https://agent-awesome.com/internal/context-snapshot \
  -o ../../../build/ai/cloudflare-context-snapshot.tar.gz
test -s ../../../build/ai/cloudflare-context-snapshot.tar.gz
gzip -t ../../../build/ai/cloudflare-context-snapshot.tar.gz
tar -tzf ../../../build/ai/cloudflare-context-snapshot.tar.gz | grep '^memory.db$'
----

If this check fails after the first successful memory write, stop the beta rollout until restore/save is fixed.

== Rollback

Redeploy the last known good revision with the same R2 bucket and secrets.
After rollback, run the Worker checks, hit `/healthz`, send one allow-listed Slack message, and verify the latest snapshot archive.
