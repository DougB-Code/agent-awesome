= Private Beta Cloudflare Deploy
:description: Cloudflare-specific deployment checklist for the private beta pilot.

This page covers the single Cloudflare Worker and Container pilot.
For per-agent provisioning, use xref:operations:private-beta-runbook.adoc[].

== Required Files

[source,sh]
----
test -f Dockerfile.cloudflare
test -f deploy/cloudflare/worker/src/index.ts
test -f deploy/cloudflare/worker/scripts/smoke-test.mjs
----

== Local Verification

[source,sh]
----
npm --prefix deploy/cloudflare/worker test
npm --prefix deploy/cloudflare/worker run check
----

The smoke test verifies Worker routing, required secrets, Container binding, R2 snapshot binding, and Slack signature forwarding without deploying.
The beta container defaults to OpenAI `gpt-5.4-mini`; only switch models by
editing `AGENTAWESOME_MODEL_ID` after confirming that the deployed OpenAI
project has access to it. The container renders the harness model config from
Worker vars on startup.

== Secrets

[source,sh]
----
cd deploy/cloudflare/worker
GATEWAY_TOKEN="$(openssl rand -base64 32 | tr '+/' '-_' | tr -d '=')"
PERSISTENCE_TOKEN="$(openssl rand -base64 32 | tr '+/' '-_' | tr -d '=')"
printf '%s\n' "$GATEWAY_TOKEN" | npx wrangler secret put AGENTAWESOME_GATEWAY_TOKEN
printf '%s\n' "$PERSISTENCE_TOKEN" | npx wrangler secret put AGENTAWESOME_PERSISTENCE_TOKEN
npx wrangler secret put OPENAI_API_KEY
----

Slack is disabled by default.
Before enabling it, set `SLACK_ENABLED` to `true` and upload `SLACK_SIGNING_SECRET` and `SLACK_BOT_TOKEN`.
Signed events from the installed Slack app route to the default profile unless profile bindings or legacy allow-list vars are configured.

== Deploy

[source,sh]
----
cd deploy/cloudflare/worker
npx wrangler r2 bucket create agent-awesome-beta-context
npx wrangler deploy
----

After Slack is enabled, set the Slack Events API request URL to:

[source,text]
----
https://agent-awesome.com/slack/events
----

Check the authenticated beta status surface after deploy:

[source,sh]
----
curl -fsS https://agent-awesome.com/healthz
curl -fsS \
  -H "Authorization: Bearer ${GATEWAY_TOKEN}" \
  https://agent-awesome.com/api/gateway/beta-status
curl -fsS \
  -H "Authorization: Bearer ${GATEWAY_TOKEN}" \
  -H "X-Agent-Awesome-Profile: doug" \
  https://agent-awesome.com/api/context/tools/list
----

== Desktop UI Check

Run the release binary against the hosted gateway:

[source,sh]
----
export AGENT_GATEWAY_BASE_URL=https://agent-awesome.com/api
export AGENTAWESOME_GATEWAY_TOKEN="${GATEWAY_TOKEN}"
export AGENTAWESOME_RUNTIME_PROFILE="<release-root>/ui/runtime_profiles/cloudflare_context.json"
export AUTO_START_LOCAL_SERVICES=false
./agentawesome_ui
----

Open chat and memory from the UI.
Both routes should use the hosted gateway, not direct memory MCP endpoints.
Use `ui/runtime_profiles/cloudflare_family.json` when testing the Family profile.

== Slack Check

After Slack Events URL verification succeeds, send one message to the installed app and inspect logs:

[source,sh]
----
npx wrangler tail agent-awesome-pilot --format pretty
----

Slack and the desktop UI should reach the same gateway-backed profile topology.
For multi-profile Slack, add `slack_bindings` to the intended profile in `AGENTAWESOME_AGENT_PROFILES_JSON`.

== Snapshot Check

[source,sh]
----
mkdir -p ../../../build/ai
curl -fsS \
  -H "Authorization: Bearer ${PERSISTENCE_TOKEN}" \
  https://agent-awesome.com/internal/context-snapshot/doug \
  -o ../../../build/ai/cloudflare-context-snapshot.tar.gz
test -s ../../../build/ai/cloudflare-context-snapshot.tar.gz
gzip -t ../../../build/ai/cloudflare-context-snapshot.tar.gz
tar -tzf ../../../build/ai/cloudflare-context-snapshot.tar.gz | grep '^memory.db$'
----

If this check fails after the first successful memory write, stop the beta rollout until restore/save is fixed.

== Rollback

Redeploy the last known good revision with the same R2 bucket and secrets.
After rollback, run the Worker checks, hit `/healthz`, send one Slack message, and verify the latest snapshot archive.
