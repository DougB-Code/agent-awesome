= Desktop UI
:description: How users start and configure the Agent Awesome desktop UI.

The desktop UI is a Flutter application in `ui`.
It starts with the default service topology and connects to the local gateway, harness, and memory services described by that topology.
The default local harness hosts the workflow endpoint and attaches command tools directly to the ADK runtime.

The default topology expects the checked-in Agent Awesome harness config under `harness`.
It also needs a model credential, such as `OPENAI_API_KEY`, available to the harness keyring or environment.

== Start The UI

From `ui`:

[source,sh]
----
flutter run -d linux
----

The default runtime topology is `runtime_topology/agent_awesome.json`.
When local service auto-start is enabled, the UI starts the services it owns and marks unavailable services as disconnected.

== Common Defines

[cols="1,2",options="header"]
|===
|Define |Use

|`AGENT_GATEWAY_BASE_URL`
|Gateway API endpoint used by the UI.

|`AGENT_API_BASE_URL`
|Harness API endpoint used by the gateway as its upstream service.

|`MEMORY_MCP_URL`
|Memory MCP endpoint used by the gateway as its upstream memory service.

|`AUTO_START_LOCAL_SERVICES`
|Whether the UI should start managed local services from the runtime topology.
|===
