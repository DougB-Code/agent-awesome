= Development Setup
:description: Reproducible workstation setup for Agent Awesome development and beta checks.

Use the pinned tool versions before running build or beta-readiness commands.
The repository declares them in `.tool-versions` so `asdf` and compatible tool managers can install the same Go and Node.js versions used by the modules.

== Toolchain

[cols="1,2,2",options="header"]
|===
|Tool |Pinned version |Why

|Go
|`1.26.0`
|Matches the Go module requirements for `gateway`, `harness`, and `memory`.

|Node.js
|`22.15.0`
|Matches the documentation build environment used by CI.
|===

Install with a `.tool-versions` aware manager:

[source,sh]
----
asdf install
----

Or install equivalent versions through your normal package manager.

Verify the active tools:

[source,sh]
----
go version
node --version
npm --version
----

== Dependencies

Install root documentation dependencies:

[source,sh]
----
npm install --cache ./build/npm-cache
----

== CI Verification

Run the documentation check from the repository root:

[source,sh]
----
npm test
----

The GitHub `CI Verification` workflow runs the full source gate for Go modules, the desktop UI, Codex pilot E2E, and docs.
