= Build And Package
:description: Verified build and packaging steps for Agent Awesome.

This page builds all local Agent Awesome artifacts and creates a Linux x64 package under `build`.
Run commands from the repository root unless a step explicitly changes directories.

== Verify Source

[source,sh]
----
cd /home/doug/dev/agentawesome/agent

npm install --cache ./build/npm-cache
npm --prefix deploy/cloudflare/worker install --cache ./build/npm-cache
npm test
(cd deploy/cloudflare/worker && npm test)
----

The GitHub `CI Verification` workflow runs Go module tests, the Cloudflare Worker smoke test, Flutter analysis/tests, the Linux release build, the Cloudflare container build, and the Antora documentation build.

`npx wrangler` commands require Node 22 or newer.
If the Worker smoke test passes but Wrangler refuses to start, update the active Node version before deploying Cloudflare resources.

== Verify Model IDs

Before beta packaging, run a live model smoke check with the provider
credentials available in the harness keyring or environment:

[source,sh]
----
cd /home/doug/dev/agentawesome/agent/harness
go run ./cmd/agent-awesome models check \
  --model model.yaml \
  --provider openai \
  --model-id gpt-mini
cd ..
----

== Build Go Binaries

[source,sh]
----
cd /home/doug/dev/agentawesome/agent
mkdir -p build/package/bin

cd harness
go build -trimpath -o ../build/package/bin/agent-awesome ./cmd/agent-awesome
cd ../memory
go build -trimpath -o ../build/package/bin/memoryd ./cmd/memoryd
cd ../gateway
go build -trimpath -o ../build/package/bin/agent-gateway ./cmd/agent-gateway
cd ../provision
go build -trimpath -o ../build/package/bin/agent-awesome-provision ./cmd/agent-awesome-provision
cd ..
----

Expected binaries:

[source,text]
----
build/package/bin/agent-awesome
build/package/bin/memoryd
build/package/bin/agent-gateway
build/package/bin/agent-awesome-provision
----

== Build The Desktop UI

[source,sh]
----
cd /home/doug/dev/agentawesome/agent/ui
flutter build linux --release
cd ..
----

Flutter writes the Linux bundle to:

[source,text]
----
ui/build/linux/x64/release/bundle
----

== Create A Package Archive

Use a fresh package directory so older artifacts are not mixed into the archive:

[source,sh]
----
cd /home/doug/dev/agentawesome/agent
PACKAGE_DIR="build/package/agent-awesome-linux-x64-$(date +%Y%m%d%H%M%S)"
mkdir -p "$PACKAGE_DIR/bin" "$PACKAGE_DIR/ui" "$PACKAGE_DIR/config/harness"

cp build/package/bin/agent-awesome "$PACKAGE_DIR/bin/"
cp build/package/bin/memoryd "$PACKAGE_DIR/bin/"
cp build/package/bin/agent-gateway "$PACKAGE_DIR/bin/"
cp build/package/bin/agent-awesome-provision "$PACKAGE_DIR/bin/"
cp -R ui/build/linux/x64/release/bundle "$PACKAGE_DIR/ui/linux-x64"
cp harness/model.yaml "$PACKAGE_DIR/config/harness/model.yaml"
cp harness/agent.yaml "$PACKAGE_DIR/config/harness/agent.yaml"
cp harness/tool.local.yaml "$PACKAGE_DIR/config/harness/tool.local.yaml"

tar -C "$PACKAGE_DIR/.." -czf "$PACKAGE_DIR.tar.gz" "$(basename "$PACKAGE_DIR")"
ls -lh "$PACKAGE_DIR.tar.gz"
----

The archive contains the Go binaries, the Flutter Linux bundle, and the default Agent Awesome harness config.
