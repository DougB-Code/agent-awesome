= Build And Package
:description: Verified build and packaging steps for Agent Awesome.

This page builds all local Agent Awesome artifacts and creates the same Linux x64 release bundle that GitHub Actions publishes.
Run commands from the repository root unless a step explicitly changes directories.

== Verify Source

[source,sh]
----
cd /home/doug/dev/agentawesome/agent

npm install --cache ./build/npm-cache
npm test
----

The GitHub `CI Verification` workflow runs Go module tests, Flutter analysis/tests, the Linux release build, Codex pilot E2E, and the Antora documentation build.

== Verify Model IDs

Before beta packaging, run a live model smoke check with the provider
credentials available in the harness keyring or environment:

[source,sh]
----
cd /home/doug/dev/agentawesome/agent/harness
go run ./cmd/agent-awesome models check \
  --model model.yaml \
  --provider openai \
  --model-id gpt-mini
cd ..
----

== Build Service Binaries

[source,sh]
----
cd /home/doug/dev/agentawesome/agent
RELEASE_ROOT="$PWD/build/release/agent-awesome-linux-x64"
PROFILE_ROOT="$RELEASE_ROOT/harness/build/profiles/agent-awesome"
BIN_ROOT="$PROFILE_ROOT/bin"

rm -rf "$RELEASE_ROOT"
mkdir -p "$BIN_ROOT"

cd harness
go build -trimpath -buildvcs=false -o "$BIN_ROOT/agent-awesome-harness" ./cmd/agent-awesome
cd ../memory
go build -trimpath -buildvcs=false -o "$BIN_ROOT/memory" ./cmd/memoryd
cd ../gateway
go build -trimpath -buildvcs=false -o "$BIN_ROOT/agent-awesome-gateway" ./cmd/agent-gateway
cd ..

touch "$PROFILE_ROOT/.prebuilt"
----

Expected binaries:

[source,text]
----
build/release/agent-awesome-linux-x64/harness/build/profiles/agent-awesome/bin/agent-awesome-harness
build/release/agent-awesome-linux-x64/harness/build/profiles/agent-awesome/bin/memory
build/release/agent-awesome-linux-x64/harness/build/profiles/agent-awesome/bin/agent-awesome-gateway
build/release/agent-awesome-linux-x64/harness/build/profiles/agent-awesome/.prebuilt
----

== Build The Desktop UI

[source,sh]
----
cd /home/doug/dev/agentawesome/agent/ui
flutter build linux --release
cd ..
----

Flutter writes the Linux bundle to:

[source,text]
----
ui/build/linux/x64/release/bundle
----

== Assemble Release Bundle

Copy the Flutter bundle, runtime profile templates, and harness configs into the release directory:

[source,sh]
----
cd /home/doug/dev/agentawesome/agent
RELEASE_ROOT="$PWD/build/release/agent-awesome-linux-x64"

cp -a ui/build/linux/x64/release/bundle/. "$RELEASE_ROOT/"
mkdir -p "$RELEASE_ROOT/ui" "$RELEASE_ROOT/harness" "$RELEASE_ROOT/gateway" "$RELEASE_ROOT/memory"
cp -a ui/runtime_profiles "$RELEASE_ROOT/ui/"
cp harness/agent.yaml harness/model.yaml harness/tool.yaml harness/tool.local.yaml "$RELEASE_ROOT/harness/"
cp -a gateway/slack-manifest.json "$RELEASE_ROOT/gateway/"

cat > "$RELEASE_ROOT/README.txt" <<'EOF'
Agent Awesome Linux x64 release bundle

Quick start:
  1. Extract this directory somewhere writable.
  2. Run ./agentawesome_ui directly.
  3. Complete the in-app model setup on first launch.

Go is not required for release users. The bundled service binaries live under harness/build/profiles/agent-awesome/bin.
EOF
----

== Create Release Archive

[source,sh]
----
cd /home/doug/dev/agentawesome/agent/build/release
tar -czf agent-awesome-linux-x64.tar.gz agent-awesome-linux-x64
sha256sum agent-awesome-linux-x64.tar.gz > agent-awesome-linux-x64.tar.gz.sha256
ls -lh agent-awesome-linux-x64.tar.gz agent-awesome-linux-x64.tar.gz.sha256
----

The archive contains the Flutter Linux bundle, prebuilt service binaries, runtime profiles, default Agent Awesome harness config, and the user-local installer.
