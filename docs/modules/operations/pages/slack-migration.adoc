= Slack Migration
:description: Operations runbook for moving the hosted gateway to a new Slack workspace.

This runbook rotates only Slack-specific values.
Model credentials, gateway tokens, and persistence tokens should remain unchanged unless you intentionally rotate them.

== Slack App Setup

Create or import the Slack app from `gateway/slack-manifest.json`.
For the Cloudflare deployment, set the Slack Events API request URL to:

[source,text]
----
https://agent-awesome.com/slack/events
----

Keep Socket Mode disabled for the hosted Cloudflare path.
Socket Mode is only for local testing.

== Rotate Secrets

From `deploy/cloudflare/worker`, upload the new Slack signing secret and bot token:

[source,sh]
----
npx wrangler secret put SLACK_SIGNING_SECRET
npx wrangler secret put SLACK_BOT_TOKEN
----

Do not paste Slack secrets into chat, documentation, or shell history.

== Update Allow Lists

Set the required non-secret allow-list values in `deploy/cloudflare/worker/wrangler.jsonc`:

[source,json]
----
"SLACK_ALLOWED_TEAM_ID": "",
"SLACK_ALLOWED_USER_ID": "",
"SLACK_ALLOWED_CHANNEL_ID": ""
----

Set all three ids before redeploying a Slack beta pilot.
The gateway refuses to start Slack ingress without a team, user, and channel allow-list.

== Redeploy And Verify

[source,sh]
----
npm run check
npx wrangler deploy --dry-run
npx wrangler deploy
----

After deployment, verify the Slack request URL and send a message in the allow-listed channel.
If Slack accepts events but the gateway ignores messages, check the allow-list values first.
