= Cloudflare Setup
:description: Operations checklist for preparing Cloudflare before Agent Awesome deployment.

This page prepares the Cloudflare account and local machine before either the manual pilot deployment or the per-agent provisioner is used.

== Required Accounts And Tools

[cols="1,2",options="header"]
|===
|Requirement |Why it is needed

|Cloudflare account on a Workers paid plan
|Cloudflare Containers are currently a paid Workers feature.

|Active Cloudflare zone
|Workers routes are attached to the zone that owns the agent hostname.

|R2 enabled on the account
|Agent Awesome stores memory snapshots in R2.

|Docker or a Docker-compatible daemon
|Wrangler builds and pushes the Container image during deploy.

|Node.js 22 or newer with npm
|The Worker package and current Wrangler require Node 22 or newer.

|Go 1.26
|The Go binaries are built from the `harness`, `gateway`, `memory`, and `provision` modules.

|Flutter with Dart 3.10 or newer
|The desktop UI package requires Dart `^3.10.4`.
|===

Check the local tools:

[source,sh]
----
node --version
npm --version
docker info
go version
flutter --version
----

Stop here if `node --version` is below `v22.0.0`.
The Worker TypeScript can be checked with older Node in some environments, but Wrangler deployment will fail.

== Cloudflare API Token

Create a Cloudflare user API token for the account and zone used by Agent Awesome.
Use the narrowest permissions that satisfy these operations:

[cols="1,2",options="header"]
|===
|Permission |Used for

|Account, `Workers Scripts Edit`
|Create, inspect, deploy, and update Worker scripts and Container-backed Workers.

|Account, `Workers R2 Storage Edit`
|Create, inspect, and delete per-agent R2 buckets.

|Zone, `Zone Read`
|Resolve the zone id by zone name.

|Zone, `DNS Read`
|Check hostname DNS readiness.

|Zone, `Workers Routes Edit`
|Create, repair, and delete Worker routes.
|===

Store the token for the provisioner:

[source,sh]
----
cd /home/doug/dev/agentawesome/agent/provision
go run ./cmd/agent-awesome-provision credentials set CLOUDFLARE_API_TOKEN
----

For manual Wrangler-only commands, either export the token in the terminal or run `npx wrangler login` from the Worker directory:

[source,sh]
----
export CLOUDFLARE_ACCOUNT_ID=<account-id>
export CLOUDFLARE_API_TOKEN=<token>
----

Do not put API tokens in `wrangler.jsonc`, `.env` files committed to git, shell history, or documentation.

== Zone And Hostnames

The current manual pilot config uses this route:

[source,text]
----
agent-awesome.com/*
----

Per-agent cloud agents derive hostnames from the platform suffix.
For agent id `sister` and suffix `agent-awesome.com`, the provisioner derives:

[source,text]
----
sister.agent-awesome.com
----

Make sure the zone is active in the Cloudflare account before running apply.
If the hostname is new, create or delegate the DNS record required by your account policy before expecting health checks to pass.
